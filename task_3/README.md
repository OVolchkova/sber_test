# Задание 3. Анализ сентимента Telegram-канала MarketTwits

Программный модуль, который:
1. Парсит Telegram-канал [@markettwits](https://t.me/markettwits) за заданный период
2. Кластеризует сообщения двумя способами
3. Выявляетинвестиционные сигналы (по новизне, аномалиям и сочетаниям тегов)
4. Считает дневной сентимент каждой публикации (negative / neutral / positive) двумя моделями (для устойчивости) и агрегирует по дням
5. Строит графики динамики дневного сентимента + сохраняет результаты в файлы.

## Архитектура

### Файлы проекта
Файл:
`parse_telegram.py` - парсер телеграмм канала с использованием proxy
`clean.py` - файл, который чистит посты от символов, отличающихся от текста
`clustering.py` - файл, который кластеризует посты двумя способами (KMeans, HDBSCAN+UMAP)
`signals.py` - достает инвестиционные сигналы
`sentiment.py` - определяет сентимент с помощью негативных и позитивных корней слов, а также с помощью модели rubert-tiny2-russian-sentiment
`visualize.py` - визуализация
`main.py` - запуск
`requirements.txt` - зависимости
`.env` - файл с ключами для входа в телеграмм (по возможности, пожалуйста, не входите в этот файл)


## Получение Telegram API-доступа

1. На https://my.telegram.org - API development tools.
2. Создать приложение - получить `api_id` (число) и `api_hash` (строка).
3. Записать в `.env` TG_API_ID, TG_API_HASH, TG_PHONE

При первом запуске Telethon отправит код подтверждения в Telegram ввести в консоли. Сессия сохранится в `markettwits.session`, повторно подтверждать не нужно.

## Запуск

### Полный пайплайн (по умолчанию - последний календарный месяц):
python main.py

### Произвольное окно:
python main.py --since 2026-03-01 --until 2026-04-01
python parse_telegram.py --since 2026-03-01 --until 2026-04-01

## Что получится на выходе

`posts.parquet` - все сообщения за период
`clusters_*.csv` - соответствие post_id - метка кластера, по одному файлу на метод
`topics_*.json` - топ-слова/представители каждого кластера
`signals.csv` - таблица потенциальных инвестиционных сигналов с колонками:
  тип сигнала, дата, текст, кластер, причина
`sentiment.parquet` - сентимент по каждой публикации

## Логика «неочевидных» сигналов

В `signals.py` реализованы три варианта сигнала:

1. **Аномалия объёма по теме**: за один день в редком кластере появляется ≥3 сообщений (норма = 1). Это часто предвещает новостной кейс (например, серия сообщений про Новатэк за один день)

2. **Связка тикер + редкое слово**: ищем co-occurrence тикеров (`#GAZP`) или просто заглавные тикеры из словаря MOEX_TICKERS с
   редкими словами из top-100 IDF (например, тикер + «расследование» или + «штраф» — сильный негативный сигнал)

3. **Резкий разворот сентимента по тикеру**: если у тикера за последние 30 дней средний сентимент был положительный, а за последние 3 дня
   средний стал отрицательный (или наоборот), это сигнал на пересмотр.

## Заметка по сентименту

Используется **двухмодельный ансамбль**:

- `seara/rubert-tiny2-russian-sentiment` (HuggingFace) - основная модель, обученная на русском.
- Лексикон: словарь негативных/позитивных слов финансовой лексики 
  (банкротство, дефолт, штраф vs дивиденды, рекорд, рост) - нужен, чтобы не съезжать на нейтральный класс на коротких сообщениях, типичных для MarketTwits.

Финальный результат получается из взвешивания 0.3 * лексикон + 0.7 * результат модели
